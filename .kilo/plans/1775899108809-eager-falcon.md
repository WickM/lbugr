# Plan: Task 1 - Update as.data.frame method for real_ladybug 0.15.x

## Problem
real_ladybug Python package version 0.15.x returns an iterator instead of a query result object when calling `get_all()`. The current `as.data.frame.ladybug.query_result.QueryResult` method in `R/lb.R` line 100 uses `x$get_all()` which returns an iterator, causing the error "cannot coerce class 'c(\"real_ladybug.query_result.QueryResult\", \"python.builtin.iterator\")' to a data.frame".

## Root Cause
The QueryResult's `get_all()` method now returns a Python iterator instead of a list. This needs to be converted to a list before processing in R.

## Solution
Convert the Python iterator to a list before processing. In reticulate, this can be done using:
- `reticulate::py_iterate(x$get_all())` - returns an R list from Python iterator
- Or `reticulate::py_as_object(x$get_all())` to convert to R list
