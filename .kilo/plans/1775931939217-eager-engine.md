# Code Review: lbugr R Package

## Summary

Thorough code review of the `lbugr` R package -- an R interface to the Ladybug graph database via reticulate/Python. The review covers all 5 source files, 5 test files, configuration, CI/CD, and packaging.

---

## Implementation Plan: Critical Issues Only

### CR-001: Fix .Rbuildignore (excludes NAMESPACE and man pages)

**File:** `.Rbuildignore`

**Problem:** Lines 3-4 (`^.*\.Rd$`, `^man\./`) exclude all .Rd man pages and the man/ directory. Line 6 (`^NAMESPACE$`) excludes the NAMESPACE file. Without these, `R CMD build` produces a broken tarball -- the package won't install.

**Action:** Rewrite `.Rbuildignore` to only exclude non-package development files. The new content should be:

```
^lbugr\.Rproj$
^\.Rproj\.user$
^Meta/
^\.github$
^codecov\.yml$
^Agent\.md$
^TESTING\.md$
^cran-comments\.md$
^\.lintr$
^\.Renviron$
^\.Rprofile$
^pkgdown$
^README\.Rmd$
^\.kilo$
^nul$
^LICENSE\.md$
```

Key deletions: `^.*\.Rd$`, `^man\./`, `^NAMESPACE$`.
Key additions: Exclude dev/CI files that shouldn't be in the source tarball (`.github`, `codecov.yml`, `Agent.md`, `TESTING.md`, etc.).

### CR-002 + CR-003: Cypher injection via unvalidated table_name

**Files:** `R/lb_load_data.R`

**Problem:** `lb_copy_from_df()` has table_name validation (line 48), but these functions do NOT:
- `lb_copy_from_file()` (line 98) — used by `lb_copy_from_csv`, `lb_copy_from_json`, `lb_copy_from_parquet`
- `lb_create_table_from_df()` (line 160) — also has unvalidated column names interpolated into Cypher
- `lb_merge_df()` (line 413) — no table_name involved but no merge_query validation either

**Action — Step 1:** Add a shared `validate_table_name()` helper at the top of `lb_load_data.R` (before `lb_copy_from_df`), and refactor `lb_copy_from_df` to use it:

```r
# Validate that a table name is a safe Cypher identifier
# Prevents Cypher injection via malformed table names
validate_table_name <- function(table_name) {
  if (!is.character(table_name) || length(table_name) != 1L ||
      !grepl("^[A-Za-z_][A-Za-z0-9_]*$", table_name)) {
    stop(
      "table_name must be a valid identifier (letters, digits, underscores only).",
      call. = FALSE
    )
  }
}
```

Then in `lb_copy_from_df()`, replace lines 47-53 with a call to `validate_table_name(table_name)`.

**Action — Step 2:** Add `validate_table_name(table_name)` as the first line of `lb_copy_from_file()` (after the function signature, line 103).

**Action — Step 3:** Add `validate_table_name(table_name)` as the first line of `lb_create_table_from_df()` (line 160, after function opens). Also validate column names:

```r
# Validate column names to prevent injection
invalid_cols <- names(df)[!grepl("^[A-Za-z_][A-Za-z0-9_]*$", names(df))]
if (length(invalid_cols) > 0) {
  stop(
    paste("Invalid column names:", paste(invalid_cols, collapse = ", ")),
    call. = FALSE
  )
}
```

**Action — Step 4:** The `lb_merge_df()` function doesn't use `table_name`, so no table validation needed there. However, add a basic type check on `merge_query`:

```r
if (!is.character(merge_query) || length(merge_query) != 1L || !nzchar(merge_query)) {
  stop("`merge_query` must be a non-empty Cypher query string.", call. = FALSE)
}
```
