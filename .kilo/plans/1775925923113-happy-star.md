# Plan: Fix VirtualAlloc Memory Errors and Graph Test Failures

## Issue Summary

1. **VirtualAlloc errors during test suite execution**: Individual tests pass, but running the full test suite causes memory accumulation and VirtualAlloc errors. This is a state accumulation issue - each test creates `:memory:` databases that aren't properly cleaned up.

2. **Graph tests failing**: Data not being returned properly from graph queries, needs debugging.

## Root Cause Analysis

### VirtualAlloc Issue
- Each test creates a new in-memory Ladybug database via `lb_connection(":memory:")`
- Tests do NOT call `db$shutdown()` to properly release resources
- Running many tests in sequence causes Python/R memory to fill up
- Eventually triggers VirtualAlloc error when trying to allocate more memory
- This is a **known behavior** - `:memory:` databases accumulate unless explicitly shut down

### Graph Data Issue
- Tests expect `as_igraph()` and `as_tidygraph()` to extract node data from query results
- The `extract_graph_data()` function in `R/lb_graph.R` relies on node/rel objects having properties like `_id`, `_label`, `_src`, `_dst`
- Need to verify what data structure is actually returned by real_ladybug 0.15.x

## Solution Plan

### Part 1: Fix VirtualAlloc - Add Proper Cleanup to Tests

**Option A: Add on.exit cleanup to each test (Recommended)**
Modify tests to add cleanup that runs even on failure:
```r
test_that("test name", {
  conn <- lb_connection(":memory:")
  on.exit({
    main <- reticulate::import_main()
    main$db$shutdown()
  }, add = TRUE)
  # ... test code
})
```

**Option B: Create a test fixture with auto-cleanup**
Update `tests/testthat/helpers.R` to have `test_conn()` function properly clean up:
```r
test_conn <- function(test = NULL) {
  conn <- lb_connection(":memory:")
  # Register cleanup to run after test completes
  test$on.exit(function() {
    main <- reticulate::import_main()
    if (!is.null(main$db)) {
      main$db$shutdown()
    }
  }, add = TRUE)
  conn
}
```
