# Plan: Stabilize Full Test Suite Execution

## Objective
Make full `devtools::test()` reliable by reducing cross-test memory pressure, enforcing a single Python runtime in local runs, and hardening cleanup behavior.

## Current State
- Graph-related tasks were already addressed and targeted graph tests pass.
- Full suite still shows intermittent VirtualAlloc under load and additional deterministic assertion/warning failures.
- Existing tests in stress-sensitive files still create many `:memory:` databases.

## Scope For This Iteration
1. Replace high-volume `:memory:` usage with per-test temp on-disk databases in:
   - `tests/testthat/test-lb.R`
   - `tests/testthat/test-lb_load_data.R`
   - `tests/testthat/test-lb_create_table_from_df.R`
2. Harden cleanup to always shutdown DB and remove temp files/directories safely.
3. Enforce single Python path in local test runs via `RETICULATE_PYTHON` checks/configuration in test harness.

## Implementation Plan
1. **Upgrade shared test helpers (`tests/testthat/helpers.R`)**
   - Introduce a robust fixture helper (e.g., `with_test_conn()` or improved `test_conn()`) that:
     - creates a unique temp test directory,
     - opens a DB path in that directory (not `:memory:`),
     - registers teardown with guarded `db$shutdown()` + `gc()` + recursive unlink,
     - tolerates partial failures during teardown.
   - Keep `cleanup_db()` but harden it:
     - prefer explicit shutdown of passed connection/main$db,
     - wrap each teardown phase in `tryCatch` to avoid masking test failures.

2. **Migrate stress-sensitive tests to fixture helper**
   - Update connection setup in the three target files to use the new disk-backed helper.
   - Replace repetitive `lb_connection(":memory:")` + `on.exit(cleanup_db(), add = TRUE)` blocks with helper-driven setup/teardown.
   - Preserve existing per-test file cleanup logic (CSV/JSON temp files), ensuring additive `on.exit(..., add = TRUE)` order is maintained.

3. **Force one Python runtime for local test execution**
   - Add test startup guard in `tests/testthat.R` (or helper loaded during tests):
     - check `Sys.getenv("RETICULATE_PYTHON")` is set,
     - fail early with clear message if unset/mismatched for local runs,
     - optionally pin to project `.Renviron` default path when empty.
   - Keep CI behavior unchanged (already sets `RETICULATE_PYTHON`).

4. **Validation sequence**
   - Run focused files first:
     - `devtools::test(filter='lb$')`
     - `devtools::test(filter='lb_load_data')`
     - `devtools::test(filter='lb_create_table_from_df')`
   - Then run full suite: `devtools::test()`.
   - Capture whether VirtualAlloc disappears from full run.

5. **Post-stabilization cleanup of remaining deterministic failures**
   - If full-suite memory issue is fixed, address remaining non-memory failures (assertion/schema/warning expectations) as separate logical fixes.

## Success Criteria
- No VirtualAlloc errors in full `devtools::test()` run on Python 3.14 local baseline.
- Stress-sensitive files no longer rely on `:memory:` in per-test setup.
- Cleanup is robust (shutdown + filesystem cleanup guarded) and does not leak state across tests.
- Python runtime path is explicitly controlled for local runs.
