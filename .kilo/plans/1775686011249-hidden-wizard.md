# Plan: Fix VirtualAlloc memory errors - Python 3.13+ compatibility issue

## Problem Summary
Testing with Python 3.13 and real_ladybug 0.15.2 causes:
```
RuntimeError: Buffer manager exception: VirtualAlloc for size 8796093022208 failed 
with error code 8: Not enough memory resources are available to process this command
```

## Root Cause Analysis

The issue is a **known bug in kuzu** (the underlying database that real_ladybug wraps):

1. **kuzu PR #4687** (merged Jan 8, 2025) - "Fix python3.13 bug" addressed this exact VirtualAlloc issue
2. **kuzu Issue #4685** - "Segmentation Fault When Using Kuzu COPY FROM with Polars DataFrame" was the original report
3. **Ladybug v0.15.0** (Feb 28, 2026) - Added **python 3.14 support** and likely includes this fix
4. **real_ladybug** is the Python wrapper for Ladybug - newer versions should work with Python 3.13+/3.14

## Solutions Available

### Option 1: Use Python 3.9-3.12 (Works, but outdated)
- The current CI already uses **Python 3.9** (see `.github/workflows/R-CMD-check.yaml:46`)
- Simple but limits you to older Python versions

### Option 2: Upgrade real_ladybug to 0.15.0+ (Recommended)
- **real_ladybug 0.15.0+** includes Python 3.14 support
- Ladybug v0.15.0 added Python 3.14 support (PR #221)
- This should fix the VirtualAlloc issue for both Python 3.13 and 3.14
- **Recommended action**: Update to real_ladybug >= 0.15.0

### Option 3: Test with Python 3.14
- The latest Python 3.14 should work with real_ladybug 0.15.0+
- This gives you the newest Python version

## Recommended Action

**Option 2**: Upgrade real_ladybug to version 0.15.0 or later, then use Python 3.14.

Steps:
1. Update CI to use Python 3.14 (in `.github/workflows/*.yaml`)
2. Update `reticulate::py_install("real_ladybug")` to install latest version (or pin to >= 0.15.0)
3. Test to confirm VirtualAlloc error is resolved
4. Update documentation to specify real_ladybug >= 0.15.0
